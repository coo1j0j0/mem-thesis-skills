# MEM Thesis Skills

这是一个面向 MEM、工程管理、项目管理、公共管理等管理类专业硕士论文写作的 Codex skill。它的目标不是替代学校模板，也不是生成一篇脱离材料的通用论文，而是帮助使用者把真实项目、组织问题和已有材料整理成符合管理类研究生论文逻辑的选题、开题报告、大纲和章节草稿。

## 解决的问题

这个 skill 主要解决以下问题：

- 论文容易写成技术说明书、项目总结或咨询报告，而不是管理类研究生论文。
- 选题边界反复漂移，题目、研究对象、研究问题和章节结构不一致。
- 开题报告容易套模板，缺少“现实问题-管理诊断-优化方案-效果验证”的主线。
- 方法设计过重，使用了 AHP、德尔菲、SEM、复杂成熟度模型等超出材料支撑能力的方法。
- 资料不足时容易编造数据、引用、访谈结论、专家意见或过强效果表述。
- 写作中容易出现 AI 味：空泛转折、宣传式表达、三段式堆叠、夸大创新和结论。
- 管理类案例论文不知道应该先收集哪些真实材料，哪些材料可作为证据，哪些内容只能标记待补充。
- 学校硬格式、开题表单和通用写作工作流容易混在一起，导致文档看似完整但不一定可提交。

## 适用场景

适合用于：

- 管理类专业硕士论文选题收敛
- MEM / 工程管理论文开题报告起草或审查
- 案例研究型论文大纲设计
- 项目管理、流程管理、组织管理、信息化管理等方向的问题诊断
- 论文材料清单和证据链整理
- 章节草稿写作与去 AI 味润色
- 导师反馈后的修改清单
- 答辩前的问题准备和解释口径整理

## 不适用场景

不适合用于：

- 自动编造数据、访谈、参考文献、专家意见或答辩结论
- 替代学校官方论文模板、封面、签字页和格式规范
- 写纯理论论文或纯计量模型论文
- 在没有材料的情况下直接生成完整可提交论文
- 对具体学校要求作最终判断，除非用户提供了学校模板或规范文件

## 使用说明

### 在 Codex 中调用

本机已安装该 skill 时，可以直接在对话里写：

```text
Use $mem-thesis-writer + 你的论文任务
```

例如：

```text
Use $mem-thesis-writer 帮我把这个管理类专硕论文题目收敛一下，说明研究对象、核心问题和不建议写入题目的内容。
```

```text
Use $mem-thesis-writer 根据我这个项目材料，先做论文证据清单，再给出 MEM 开题报告大纲。
```

```text
Use $mem-thesis-writer 审查我的开题报告，按“必须补齐、建议修改、可最后优化”列出问题。
```

```text
Use $mem-thesis-writer 把这一段论文文字去 AI 味，但不要增加事实、数据或引用。
```

### 常见任务模板

选题收敛：

```text
Use $mem-thesis-writer 基于我现在的材料，把题目、研究对象、核心问题和论文边界重新收敛。不要把PMBOK/PDCA/ITIL写进题目，输出“推荐题目、研究对象、核心问题、论文边界、下一步需核实材料”。
```

开题报告审查：

```text
Use $mem-thesis-writer 按学校开题报告要求审查这份材料，先列“必须补齐、建议修改、可最后优化”，每条说明风险和具体改法，不要直接重写全文。
```

论文大纲：

```text
Use $mem-thesis-writer 按MEM案例分析论文逻辑生成三级目录。论文要保持甲方视角，写管理问题而不是技术实现，方法简化，效果验证指标不要太多。
```

证据材料清单：

```text
Use $mem-thesis-writer 根据我的论文题目和本地材料，列出优先收集的项目资料、过程记录、访谈问题和可用表格，并区分“已确认、可补充、访谈/问卷可验证、不可写实”。
```

章节草稿：

```text
Use $mem-thesis-writer 写“问题识别与成因分析”这一节，只依据我给出的材料。缺失信息用“待补充”标记，不要添加虚构数据、引用或专家意见。
```

去 AI 味润色：

```text
Use $mem-thesis-writer 对下面文字做论文式润色，保留事实和范围，去掉宣传式表达、空泛转折和过强结论，不增加新观点。
```

SWJTU 硕士论文格式检查：

```text
Use $mem-thesis-writer 按西南交通大学硕士论文格式检查这份论文，重点看封面/扉页、摘要关键词、目录层级、字体段落、图表、参考文献、页眉页码和装订顺序。输出“必须按模板处理、格式风险、可写作优化、待补充字段”。
```

### 输出使用建议

- 做选题、目录、开题报告时，先让 skill 输出诊断和清单，再决定是否重写正文。
- 做 SWJTU 格式检查时，应同时提供学校原始模板或当前 Word 文档；固定声明、签字页、扉页版式以学校模板为准。
- 做章节草稿时，先提供真实材料来源；没有材料的地方让 skill 标 `待补充`，不要让它补成事实。
- 做去 AI 味时，给出原文和约束，例如“不要改变题目”“不要增加指标”“不要写成咨询报告”。
- 做参考文献或引用清理时，要求区分“本地可核验引用”和“待补充来源”，避免虚空引用。

## 输出风格

默认输出应当：

- 先确认题目、研究对象、核心问题和材料边界
- 用管理问题而不是技术细节组织论文
- 采用案例分析逻辑
- 保持方法简洁，优先使用文献研究、案例研究、访谈/问卷辅助、过程分析等可落地方法
- 对缺失材料使用 `待补充` 或 `待按学校模板确认`
- 避免夸大效果、虚构引用和宣传式结论
- 使用自然、克制、可放入论文的中文学术表达

## 与学校模板的关系

这个 skill 的主线仍然是管理类论文写作和审查工作流，不把通用写作逻辑伪装成学校硬格式。

当前已内置一份来自用户本地 `扉页及范式—硕士.docx` 的西南交通大学硕士论文格式拆解。遇到 SWJTU/MEM 硕士论文封面、扉页、摘要、目录、字体、参考文献、页眉页码、装订顺序等问题时，应先读取 `references/swjtu-master-thesis-format.md`；涉及版式、签字页和固定声明文字时，应回到学校原始 `.docx` 模板，不要只依赖文字摘要。

如果学校后续发布更新版本，应以研究生院最新模板和规范文件为最终依据。
